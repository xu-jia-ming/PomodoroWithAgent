# 技术方案与架构

## 1. 总体架构
- 前端（`frontend`）：Vue3 + Vite + Element Plus + Pinia + Vue Router
- 后端（`backend`）：FastAPI + Pydantic
- 后端环境管理：Conda（环境名建议 `pomodoro-table`）
- 通信：HTTP REST API（JSON）

## 2. 目录规划
```text
PomodoroTable/
  docs/
  frontend/
  backend/
```

## 3. 前端设计
- `App.vue`：主容器，底部 Tab 导航
- `views/`：
  - `TodosView.vue`
  - `CollectionsView.vue`
  - `FocusLockView.vue`
  - `StatsView.vue`
  - `ProfileView.vue`
- `api/`：封装请求
- `stores/`：状态管理（后续扩展）

## 4. 后端设计
- `main.py`：应用入口
- `desktop_server.py`：桌面打包入口（用于生成内置后端 exe）
- `schemas.py`：数据模型
- `services.py`：业务逻辑（SQLite 持久化）
- `routes.py`：路由定义

## 5. 数据模型（MVP）
- Todo：id, title, estimated_pomodoros, completed, collection_id
- Collection：id, name
- FocusSession：duration_minutes, active, end_time
- Stats：today_pomodoros, week_pomodoros, total_focus_minutes
- User：id, nickname

## 5.1 当前数据存储说明（MVP）
- 当前已接入 SQLite 本地数据库
- 开发模式数据库文件：`backend/data/pomodoro.db`
- 桌面安装包数据库文件：`%APPDATA%/PomodoroTable/backend-data/pomodoro.db`
- 服务层实现：`backend/app/services.py` 的 `SQLiteService`
- 后端重启后，待办、统计、会话记录会保留

## 5.2 Windows 桌面打包链路
- `frontend/npm run desktop:build` 会先构建前端，再构建后端 exe，再由 electron-builder 打包。
- 后端 exe 由 `backend/desktop_server.py` + PyInstaller 生成，产物：`backend/dist-desktop/pomodoro-backend.exe`。
- 桌面应用启动时会自动拉起内置后端，前端默认请求 `http://127.0.0.1:8000`。

## 6. 演进策略
- MVP 使用 SQLite 本地持久化，便于快速验证
- 下一阶段可升级为 PostgreSQL
- Android 阶段复用后端 API，前端替换为移动端实现
