# 开发与启动指南（Windows + Android）

## 1. 环境要求
- Python 3.11+
- Conda（Miniconda 或 Anaconda）
- Node.js 20+
- PyInstaller（用于桌面打包，可通过 `pip install -r backend/requirements-desktop.txt` 安装）

## 2. 后端启动（FastAPI + Conda）
在项目根目录执行：

```bash
cd backend
conda create -n pomodoro-table python=3.11 -y
conda activate pomodoro-table
pip install -r requirements.txt
uvicorn main:app --reload --host 127.0.0.1 --port 8000
conda activate pomodoro-table; uvicorn main:app --reload --host 127.0.0.1 --port 8000
```

后端健康检查：
- 打开 `http://127.0.0.1:8000/`

## 3. 前端启动（Vue3 + Element Plus）
另开一个终端，在项目根目录执行：

```bash
cd frontend
npm install
npm run dev
```

前端访问地址：
- `http://127.0.0.1:5173`

功能快速验证：
- 进入“待办”页，顶部“番茄倒计时”点击开始后应每秒递减
- 在“待办”页新增一个待办后，应自动开始该待办的番茄倒计时
- 倒计时结束后应提示完成，统计页“今日番茄/本周番茄/专注分钟”会更新
- 待办列表里“执行”可对已完成待办再次发起计时
- 计时模式可切换“倒计时/正计时”
- 页面顶部可切换 “Windows UI / Android UI” 预览

## 4. 常见问题
### 4.1 你之前执行了 `npm run serve` 失败
本项目基于 Vite，启动命令应为：

```bash
npm run dev
```

### 4.2 Conda 未生效
先初始化 shell：

```bash
conda init powershell
```

然后重开 PowerShell 再执行 `conda activate pomodoro-table`。

### 4.3 数据库和数据保存位置
- 当前使用 SQLite 本地数据库
- 开发模式数据路径：`backend/data/pomodoro.db`
- 桌面安装包模式数据路径：`%APPDATA%/PomodoroTable/backend-data/pomodoro.db`
- 后端重启后数据会保留；删除对应数据库文件后数据会重置
- 当数据库为空时，系统会自动写入一批默认待办和最近一周历史会话数据，用于观察统计曲线

## 5. Windows 桌面应用打包（Electron）
在项目根目录执行：

```bash
cd backend
conda activate pomodoro-table
pip install -r requirements-desktop.txt
cd frontend
npm install
npm run desktop:build
```

打包输出目录：
- 安装包：`frontend/release/PomodoroTable-Setup-0.1.0.exe`
- 免安装目录：`frontend/release/win-unpacked/`
- 后端可执行文件（中间产物）：`backend/dist-desktop/pomodoro-backend.exe`

图标资源：
- Windows 安装包与快捷方式图标：`frontend/build/icon.ico`
- Web/Favicon 图标：`frontend/public/favicon.ico`
- 高分辨率源图：`frontend/public/icon-256.png`

### 5.1 使用代理打包（127.0.0.1:7890）
```powershell
$env:HTTP_PROXY="http://127.0.0.1:7890"
$env:HTTPS_PROXY="http://127.0.0.1:7890"
$env:ELECTRON_GET_USE_PROXY="true"
$env:ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"
npm run desktop:build
```

### 5.2 打包后如何使用
1. 双击安装包 `PomodoroTable-Setup-0.1.0.exe` 完成安装。
2. 启动桌面客户端，应用会自动拉起内置 FastAPI 后端（默认地址 `http://127.0.0.1:8000`）。
3. 验证待办新增、倒计时和统计页数据更新是否正常。
4. 如需改为远程 API，可设置环境变量 `POMODORO_API_BASE_URL` 后再启动应用。

### 5.3 替换应用图标
1. 准备新的方形 PNG（建议至少 256x256）。
2. 覆盖 `frontend/public/icon-256.png`。
3. 重新生成 `frontend/build/icon.ico` 与 `frontend/public/favicon.ico`。
4. 执行 `npm run desktop:build` 重新打包。

## 6. 发布前检查清单（Windows）
- 内置后端可被桌面端自动拉起并访问 `http://127.0.0.1:8000/docs`
- 前端可执行 `npm run build` 成功
- 桌面包可执行 `npm run desktop:build` 成功
- 安装包可在干净用户目录完成安装与启动
- 核心流程可用：新增待办、启动番茄钟、查看统计

## 7. Android 打包（Vue + FastAPI）
Android 打包在 `feature/mobile-packaging` 分支推进，建议先并入 Windows 打包分支：

```bash
git checkout feature/mobile-packaging
git merge --no-ff feature/windows-packaging
```

推荐方案：
- 前端使用 Capacitor 打包为 Android WebView 容器。
- 后端 FastAPI 使用内嵌 Python 运行时在手机本机启动（`127.0.0.1:8000`）。

在 `frontend` 目录可直接执行：

```bash
npm run mobile:sync
npm run mobile:build:debug
```

详细步骤、Gradle/Chaquopy 配置、签名发布流程见：
- `docs/07-Android打包指南.md`
